# 🚀 Quick Deployment to Vercel + Railway

## Step 1️⃣: Frontend sur Vercel (5 min)

```bash
# 1. Go to vercel.com → Sign up with GitHub
# 2. Click "Add New" → "Project"
# 3. Select: FULLALLSTARSBATTLE repo → Root: ALLSTARSBATTLE directory
# 4. Build settings are auto-detected from vercel.json
# 5. Add Environment Variable:
#    VITE_API_URL = https://allstarsbattle-api.up.railway.app/api
# 6. Click Deploy → Wait 2-3 minutes ✅
```

**URL**: `https://allstarsbattle.vercel.app`

---

## Step 2️⃣: Backend sur Railway (10 min)

```bash
# 1. Go to railway.app → Sign up with GitHub
# 2. Click "New Project" → Create a MySQL database
# 3. Click "New" → "GitHub Repo"
#    Select: FULLALLSTARSBATTLE/allstarsbattle-api
# 4. Railway auto-detects Dockerfile ✅
# 5. Configure Environment Variables:
```

### Railway Variables to Add:

```env
# Auto from MySQL (copy these):
DB_HOST=mysql.railway.internal
DB_PORT=3306
DB_DATABASE=railway
DB_USERNAME=root
DB_PASSWORD=<auto-generated>

# Add these:
APP_ENV=production
APP_DEBUG=false
APP_KEY=<generate: php artisan key:generate>
APP_URL=https://<your-railway-domain>.up.railway.app

MAIL_MAILER=smtp
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-specific-password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=your-email@gmail.com
MAIL_FROM_NAME=All Star Battle

CLOUDFLARE_R2_ACCOUNT_ID=your-id
CLOUDFLARE_R2_ACCESS_KEY_ID=your-key
CLOUDFLARE_R2_SECRET_ACCESS_KEY=your-secret
CLOUDFLARE_R2_BUCKET_NAME=allstarsbattle-media
CLOUDFLARE_R2_PUBLIC_URL=https://pub-xxxxx.r2.dev

FRONTEND_URL=https://allstarsbattle.vercel.app
```

```bash
# 6. Wait for deploy ✅
# 7. Get your Railway domain from Settings
# 8. Run migrations: railway run php artisan migrate
```

---

## Step 3️⃣: Connect Frontend ↔ Backend

```bash
# 1. Copy your Railway domain: https://allstarsbattle-api.up.railway.app
# 2. Go to Vercel → Project Settings → Environment Variables
# 3. Update VITE_API_URL = https://allstarsbattle-api.up.railway.app/api
# 4. Trigger redeploy (or just wait for auto-redeploy)
```

## 🚨 En Cas de Problème

**Vercel 404?**
```
→ vercel.json rewrites est en place ✅
```

**Railway 502?**
```
→ Vérifier logs: Metrics → Logs
→ Vérifier variables d'env manquantes
```

**CORS Error?**
```
→ Vérifier cors.php allows frontend domain
→ Vérifier FRONTEND_URL in .env
```

**Images manquantes?**
```
→ Vérifier Cloudflare R2 credentials
→ Vérifier CLOUDFLARE_R2_PUBLIC_URL
```
